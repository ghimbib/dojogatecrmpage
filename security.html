Security

Last Updated: April 9, 2026

Security matters more than security theater. DojoGate uses practical controls to protect customer access, data, and workflow integrity.

Current Security Practices

  • HTTPS/TLS for data in transit
  • Salted password hashing and authenticated account access
  • Role-based access controls inside the application
  • Operational review of application behavior, access, and deployment changes

Responsible Disclosure

If you believe you found a vulnerability, email support@dojogate.ai with the subject line Security. Include steps to reproduce, impact, and any proof you can share safely.

We’d rather get a crisp report with evidence than a vague “your site might be insecure” drive-by.

Need a security conversation?

If you are evaluating DojoGate for your team and need security detail during review, reach out and we’ll handle it directly.
